# AlphaHoldem 系统架构设计文档 (V1.0)

> **核心定位**：基于“链上资金结算 + 链下 TEE 机密执行”的混合 Web3 架构。
> **解决痛点**：消除纯链上扑克的高延迟与高 Gas，打破中心化平台的信任黑盒（防挪用资金、防偷看底牌）。
> **技术愿景**：打造支持从微注额 Wasm 逻辑对战，到高额桌 LLM 实时博弈的去中心化智力竞技网络。

---

## 一、 核心架构矩阵总结

| 模块名称 | 核心技术栈 | 职责界定 | 数据落点 |
| --- | --- | --- | --- |
| **结算协议 (Protocol)** | Rust / Solana Anchor | 资金托管、筹码兑换、前向种子承诺 | 链上 (Solana) |
| **机密引擎 (TEE Engine)** | .NET 10 / Wasmtime | 公平发牌、状态机流转、算力沙盒隔离 | 链下 (TEE黑盒) |
| **数据基建 (DA Archive)** | IPFS / Pinata API | 牌谱快照归档、AI 对局数据集构建 | 链下 (IPFS网络) |
| **大厅雷达 (Tracker)** | Go / Node.js / WSS | 链上事件监听、牌桌状态发现与广播 | 链下 (中心化) |
| **视觉终端 (Client)** | Unity 3D / WebGL | 3D 场景渲染、对战通信、钱包签名交互 | 用户本地终端 |
| **开发者脚手架 (CLI)** | Rust CLI | 策略模板初始化、本地沙盒调试 | 开发者本地环境 |

## 二、 模块详细拆解

### 1. 链上结算与确权协议 (`ahc-protocol`)

系统的**绝对信任层**与资金保险箱，通过智能合约锁定所有价值流转。

* **金库管理 (Vault)**：管理全局或单桌的 AHC 代币流动性，无私钥不可挪用。
* **筹码进出 (Buy-in / Cash-out)**：处理玩家上桌资金锁定，及离桌时基于 TEE 签名的资金释放。
* **规则锚定 (TableState)**：记录单桌的盲注级别、最大人数与允许的算力类型。
* **前向承诺 (Commit-Reveal)**：发牌前将 TEE 的随机种子 Hash 上链锁死；局后公布明文以供全网校验。

### 2. 机密计算对战引擎 (`enclave-dealer`)

AlphaHoldem 的**物理黑盒**，确保发牌与裁决的绝对公正。

* **公平发牌**：使用 CSPRNG 生成强加密随机种子完成洗牌。
* **高频状态机**：维护座位流转、筹码计算，执行 3 秒硬性时钟 (Shot Clock) 超时弃牌逻辑。
* **算力沙盒隔离**：
* 安全挂载 `.wasm` 策略包。
* 组装牌局上下文 JSON，调用外部大模型，处理增量状态更新 (JSON Patch)。


* **裁决与签名**：比对最终牌力，生成包含赢家信息的 Payload，使用 TEE 内部私钥签名并发送至合约提款。

### 3. 数据可用性归档层 (Data Availability)

系统的**自证清白机制**与 AI 数据源泉。

* **牌谱快照**：每局结束后，将玩家动作、底牌及明文 Seed 打包为标准 JSON。
* **异步固化**：将 JSON 异步抛入 IPFS 网络，获取唯一的 CID。
* **链上指纹**：将 CID 刻入 Solana 合约，实现极低成本的永久审计追踪。
* **开源数据集**：为全网 Wasm 与 LLM 机器人提供无门槛的真实对局训练语料。

### 4. 大厅雷达与视觉终端

负责信息分发与用户交互体验，隔离重度计算与展示层。

* **大厅雷达 (`ahc-tracker`)**：实时监听链上建桌事件，维护活跃牌桌内存列表，通过 WebSocket 向前台广播房间路由信息。
* **视觉终端 (`showdown-client`)**：集成 Phantom 钱包，渲染赛博风格的 3D 牌桌及筹码特效，与 TEE 引擎直连实现极速对战。

---

## 三、 核心数据流转时序 (Workflow)

1. **建桌发现**：`ahc-tracker` 扫描 Solana 链上，发现由 `enclave-dealer` 托管的新桌子，并推送给 Unity 客户端。
2. **上桌质押**：玩家在 Unity 唤起钱包签署交易，`ahc-protocol` 将玩家对应的 AHC 锁入合约金库。
3. **接管发牌**：`enclave-dealer` 监听到入金成功，将玩家加入座位。达到开局人数后，生成 Seed 洗牌，并将 Hash 提交至链上完成“前向承诺”。
4. **高频博弈**：TEE 向各玩家推送底牌，轮流触发 Wasm 或 LLM 策略。加注/跟注等动作通过 WSS 在链下毫秒级完成。
5. **清算归档**：牌局结束，TEE 比牌并算出赢家；将本局明文 Seed 及完整动作 JSON 传至 IPFS 获取 CID；最后将赢家数据及 CID 使用私钥签名，发送至链上合约完成真实 AHC 结算与数据上链。
